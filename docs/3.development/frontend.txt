Act as a Senior Frontend Developer.

Based on the following user stories, implement the frontend for student registration and student login.

User Story 1:
Title: Student Registration

As a student,
I want to register for the Library Management System,
So that I can borrow books and manage my library activities.

Registration Requirements:

* Full Name
* Email Address
* Student ID
* Password
* Email must be unique
* Password must contain at least 8 characters
* Registration confirmation message should appear

User Story 2:
Title: Student Login

As a registered student,
I want to log in to the Library Management System,
So that I can access my account, borrow books, and manage my library activities.

Login Requirements:

* Email Address
* Password
* Email and password must be validated
* Invalid login credentials should show an error message
* Successful login should redirect the student to the dashboard
* Student session should be securely maintained after login

Technology Stack:

* NextJS
* Material UI
* Axios

Generate:

1. UI Wireframes for registration and login
2. React Component Structure
3. Registration Form UI
4. Form Validation Rules
5. Login Form UI
6. API Integration
7. Loading States
8. Error Handling
9. Success Message and Redirect Flow
10. Auth Session Handling
11. Folder Structure
12. Complete React Component Code

Registration Form Fields:

* Full Name
* Email Address
* Student ID
* Password

Login Form Fields:

* Email Address
* Password

Frontend Behavior:

* Registration page submits to POST /api/auth/register.
* Login page submits to POST /api/auth/login.
* Disable submit buttons while API requests are loading.
* Show field-level validation errors before submitting.
* Show API error messages for duplicate registration email, duplicate student ID, and invalid login credentials.
* On successful registration, display a confirmation message and provide a path to login.
* On successful login, store the authenticated session according to the backend response strategy and redirect to /dashboard.
* Use accessible labels, helper text, and keyboard-friendly form controls.
* Keep the UI responsive for mobile, tablet, and desktop screens.

Follow modern React best practices and responsive design principles.
